
Парадигма разработки Frontend'а на Gulp.

Gulp - сборщик проекта - выполняет задачи по автоматизации работы с файлами.



1. Gulp - автоматизирует задачи по работе с файлами.

2. Файлы имеют 3 состояния: 
  :prod  (готовые для боевого сервера)
  :dev   (скомпилированные для разработки, для браузера )
  :src   (исходные файлы в которых разработчик пишет код)

3. Файловое Пространство разделено на 3 зоны
  /dist  (для :prod)
  /dev   (для :dev)
    /_scr  (для :src)

4. Структура во всех зонах одинакова (dev/ повторяет _src/, dist/ повторяет dev/)

5. Gulpfile.js - написан в функциональном стиле, чтобы делать код масштабируемым и инкапсулируемым.

6. Имена ф-й преобразующих файлы и 1-го состояния (src) во 2-е (dev) заканчиваются суффиксом -Dev,
имена ф-й преобразующих файлы из 2-го состояния в 3-е (prod) заканчиваются суффиксом -Build,
